API Documentation

REST API for the Food Ordering App, powered by Supabase. All endpoints require a Bearer token unless marked public.

BASE URL https://your-project.supabase.co/rest/v1
🔐 Authentication
Handled by Supabase Auth. Use the Supabase client SDK on the frontend — these are the underlying API calls it makes.
POST /auth/v1/signup Register a new user ▶
Creates a new user in Supabase Auth. After signup, a row in profiles should be created via a Supabase database trigger.
Request Body
FieldTypeRequiredDescription
emailstringrequiredUser's email address
passwordstringrequiredMin 6 characters
Request Body
{
  "email": "juan@example.com",
  "password": "securepassword123"
}
Response 200
{
  "access_token": "eyJhbGci...",
  "token_type": "bearer",
  "user": {
    "id": "uuid",
    "email": "juan@example.com"
  }
}
Responses
200User created, returns access token
400Email already registered or invalid input
POST /auth/v1/token?grant_type=password Sign in with email & password ▶
Authenticates a user and returns a JWT access token to use in subsequent requests.
Request Body
{
  "email": "juan@example.com",
  "password": "securepassword123"
}
Responses
200Returns access_token and refresh_token
400Invalid credentials
POST /auth/v1/logout Sign out current user ▶
Invalidates the current session token.
🔒 Requires Authorization: Bearer <token> header.
Responses
204Successfully signed out
401Missing or invalid token
👤 Profiles
User profile data extending Supabase auth. All endpoints require authentication.
GET /profiles?id=eq.{user_id} Get a user profile ▶
Returns profile data for a given user. Use auth_id=eq.{supabase_uid} to fetch by auth ID.
🔒 Requires Bearer token.
Response 200
[{
  "id": "uuid",
  "auth_id": "uuid",
  "full_name": "Juan dela Cruz",
  "role": "buyer",
  "phone": "+639171234567",
  "avatar_url": "https://...",
  "is_seller": false,
  "created_at": "2025-01-01T00:00:00Z",
  "updated_at": "2025-01-01T00:00:00Z"
}]
Responses
200Profile array (empty if not found)
401Unauthorized
PATCH /profiles?id=eq.{user_id} Update profile ▶
Updates profile fields. Only the authenticated user can update their own profile (enforced via RLS).
🔒 Requires Bearer token. Row Level Security enforced.
Request Body (all optional)
FieldTypeRequiredDescription
full_namestringoptionalDisplay name
phonestringoptionalContact number
avatar_urlstringoptionalURL from Supabase Storage
is_sellerbooleanoptionalPromote user to seller role
Responses
200Updated profile returned
403Cannot update another user's profile
🏪 Seller Profiles
Manage seller store information. A user must have is_seller: true in their profile first.
GET /seller_profiles List all sellers ▶
Returns all seller profiles. Public endpoint. Filter by is_open=eq.true to show only open stores.
Query Params
ParamTypeDescription
is_openbooleanFilter by open status e.g. is_open=eq.true
selectstringSelect specific columns e.g. select=id,store_name,is_open
Responses
200Array of seller profiles
POST /seller_profiles Create seller profile ▶
Creates a seller profile linked to the authenticated user's profile.
🔒 Requires Bearer token. User must have is_seller: true.
Request Body
FieldTypeRequiredDescription
profile_iduuidrequiredFK to profiles.id
store_namestringrequiredDisplay name of the store
descriptionstringoptionalShort store description
addressstringrequiredStore physical address
is_openbooleanoptionalDefault: false
Responses
201Seller profile created
400Missing required fields
403User is not a seller
PATCH /seller_profiles?id=eq.{seller_id} Update seller profile ▶
Update store info or toggle open/closed status.
🔒 Requires Bearer token. Only the store owner can update.
Request Body (all optional)
FieldTypeDescription
store_namestringNew store name
descriptionstringUpdated description
addressstringUpdated address
is_openbooleanToggle store open/closed
Responses
200Updated seller profile
403Not the store owner
🍔 Menu Items
Manage food items per seller. Listing is public; creating/editing requires seller auth.
GET /menu_items?seller_id=eq.{seller_id} List menu items for a seller ▶
Returns all menu items for a given seller. Add &is_available=eq.true to show only available items.
Response 200
[{
  "id": "uuid",
  "seller_id": "uuid",
  "name": "Chicken Burger",
  "description": "Crispy fried chicken sandwich",
  "price": 150.00,
  "category": "Burgers",
  "is_available": true,
  "image_url": "https://...",
  "created_at": "2025-01-01T00:00:00Z",
  "updated_at": "2025-01-01T00:00:00Z"
}]
Responses
200Array of menu items
POST /menu_items Create a menu item ▶
Adds a new food item to the seller's menu.
🔒 Requires Bearer token. Must be the seller who owns this menu.
Request Body
FieldTypeRequiredDescription
seller_iduuidrequiredFK to seller_profiles.id
namestringrequiredItem name
descriptionstringoptionalItem description
pricenumericrequiredPrice in PHP
categorystringoptionale.g. "Burgers", "Drinks"
is_availablebooleanoptionalDefault: true
image_urlstringoptionalURL from Supabase Storage
Responses
201Menu item created
400Missing required fields
403Not authorized as seller
PATCH /menu_items?id=eq.{item_id} Update a menu item ▶
Update item details. Commonly used to toggle availability or update price.
🔒 Requires Bearer token. Only the owning seller can update.
Responses
200Updated menu item
403Not the item owner
DELETE /menu_items?id=eq.{item_id} Delete a menu item ▶
Permanently removes a menu item. Consider using is_available: false instead to preserve order history references.
🔒 Requires Bearer token. Only the owning seller can delete.
Responses
204Successfully deleted
403Not the item owner
📦 Orders
Place and manage food orders. Buyers place orders, sellers update status.
GET /orders?buyer_id=eq.{user_id} List my orders (as buyer) ▶
Returns all orders placed by the authenticated buyer. Use seller_id=eq.{id} instead to get orders received as a seller.
🔒 Requires Bearer token.
Response 200
[{
  "id": "uuid",
  "buyer_id": "uuid",
  "seller_id": "uuid",
  "buyer_address_id": "uuid",
  "total_amount": 360.00,
  "status": "pending",
  "created_at": "2025-01-01T00:00:00Z",
  "updated_at": "2025-01-01T00:00:00Z"
}]
Responses
200Array of orders
401Unauthorized
GET /orders?id=eq.{order_id}&select=*,order_items(*) Get order with items ▶
Returns a single order with all its line items using Supabase's nested select.
🔒 Requires Bearer token. Only buyer or seller of that order can access.
Response 200
{
  "id": "uuid",
  "status": "confirmed",
  "total_amount": 360.00,
  "order_items": [
    {
      "id": "uuid",
      "menu_item_id": "uuid",
      "quantity": 2,
      "unit_price": 150.00
    }
  ]
}
Responses
200Order with nested items
404Order not found
POST /orders Place an order ▶
Creates an order. After creating the order, POST each item to /order_items with the returned order ID. Consider wrapping both in a Supabase Edge Function for atomicity.
🔒 Requires Bearer token.
Request Body
FieldTypeRequiredDescription
buyer_iduuidrequiredFK to profiles.id
seller_iduuidrequiredFK to seller_profiles.id
buyer_address_iduuidrequiredFK to buyer_address.id
total_amountnumericrequiredSum of all item totals
statusenumoptionalDefault: "pending"
Responses
201Order created, returns order with ID
400Missing fields or invalid address
PATCH /orders?id=eq.{order_id} Update order status ▶
Used by sellers to progress an order through statuses.
🔒 Requires Bearer token. Only the receiving seller can update status.
Status Flow
StatusDescription
pendingOrder placed, awaiting seller confirmation
confirmedSeller accepted the order
preparingFood is being prepared
readyReady for pickup/delivery
deliveredOrder completed
cancelledOrder cancelled by buyer or seller
Request Body
{
  "status": "confirmed"
}
Responses
200Updated order
403Not authorized to update this order
📍 Buyer Addresses
Saved delivery addresses per buyer. A buyer can have multiple addresses.

Returns all saved delivery addresses for the authenticated buyer.
🔒 Requires Bearer token.

Responses
200Array of addresses
401Unauthorized
POST /buyer_address Add a new address ▶
Saves a new delivery address for the buyer.
🔒 Requires Bearer token.
Request Body
FieldTypeRequiredDescription
buyer_iduuidrequiredFK to profiles.id
streetstringrequiredStreet address
barangaystringrequiredBarangay
citystringrequiredCity
countrystringoptionalDefault: "Philippines"
postal_codestringoptionalPostal/ZIP code
latitudefloatoptionalFor map/delivery features
longitudefloatoptionalFor map/delivery features
Responses
201Address saved
400Missing required fields
DELETE /buyer_address?id=eq.{address_id} Delete an address ▶
Removes a saved address. Cannot delete an address that is referenced by an existing order.
🔒 Requires Bearer token. Only the address owner can delete.
Responses
204Successfully deleted
409Address is in use by an existing order
⭐ Reviews
Post-order reviews. One review per order, only by the buyer of that order.
POST /reviews Create a review ▶
Submits a review for a completed order. Only allowed when order status is delivered.
🔒 Requires Bearer token. Only the buyer of that order can review.
Request Body
FieldTypeRequiredDescription
order_iduuidrequiredFK to orders.id
reviewer_iduuidrequiredFK to profiles.id
ratingintrequired1 to 5
commentstringoptionalWritten review (nullable)
Responses
201Review created
400Order not delivered yet or already reviewed
403Not the buyer of this order
GET /reviews?order_id=eq.{seller_id} Get reviews for a seller ▶
To get all reviews for a seller, join through orders: /reviews?select=*,orders!inner(seller_id)&orders.seller_id=eq.{seller_id}. Public endpoint.
Response 200
[{
  "id": "uuid",
  "order_id": "uuid",
  "reviewer_id": "uuid",
  "rating": 5,
  "comment": "Great food, fast delivery!",
  "created_at": "2025-01-01T00:00:00Z"
}]
Responses
200Array of reviews
